# Fast Secure Hook Review Engine for Pi/OMP

## Summary

Add a daemon-resident, typed hook review engine that targets ~200ms p95 for safe source-file reads while preserving local-first security.

## Changes

### New modules

- `runtime/hook_review_types.py` — `HookReviewRequest`/`HookReviewResponse` typed API with `to_harness_json()` serialization
- `runtime/hook_content_scanner.py` — bounded streaming secret scanner wrapping `classify_secret_content()` with byte/match/deadline budgets and rolling context windows
- `runtime/source_paths.py` — shared source-path classification refactored from CLI `commands_support_codex_paths.py` (old functions kept as thin wrappers)
- `runtime/hook_decision_cache.py` — exact source-read cache wrapper around existing `scanner_cache` with config/policy fingerprint invalidation
- `runtime/hook_source_read.py` — source-read fast-path evaluator with TOCTOU guard (stat-before/read/stat-after), output hash verification, symlink/sensitive-path rejection
- `runtime/hook_review_engine.py` — resident decision core with fail-safe deny/block on any exception
- `daemon/hook_worker.py` — resident HTTP payload handler that avoids CLI startup cost
- `daemon/hook_metrics.py` — in-memory metrics recorder (stores only buckets and counters, never raw content)

### Modified files

- `runtime/secret_sensitivity.py` — added `secret_content_rule_version()` for cache invalidation
- `cli/commands_support_codex_paths.py` — path functions refactored to delegate to `runtime/source_paths.py`
- `store_policy.py` — added `policy_fingerprint()` for cache invalidation
- `daemon/server.py` — `_handle_runtime_hook` split into fast worker + legacy CLI paths behind `HOL_GUARD_HOOK_FAST_PATH=1` feature flag

## Security

- Never trusts `guard_source_ref` alone; re-reads, re-stats, re-hashes files
- Never allows original output without exact hash match
- Never calls LLM or network in the hot path
- Fail-safe deny/block on timeout, exception, cache miss, scanner budget exhaustion
- Cache invalidates on content, stat, policy, config, or scanner version changes
- Metrics/receipts contain no raw output, prompt text, or secret samples
- Legacy CLI path unchanged when feature flag is disabled

## Tests

144 tests pass across all new modules. Existing Pi adapter, payload reference, and daemon surface server tests continue passing.
